# Synthetic Data Distillation: Teacher-Student OCR

Fine-tuning small, domain-specific OCR models using labels generated by a large vision-language model — a data-augmentation-based knowledge distillation approach.

## Overview

A 235B-parameter VLM (Qwen3-VL-235B via AWS Bedrock) acts as a **teacher**, labeling 129K table-cell images. Two smaller TrOCR models act as **students**, trained on those labels to achieve near-identical accuracy at a fraction of the cost.

```
Qwen3-VL-235B (Teacher)
  └── AWS Bedrock Batch Inference → 129,041 labeled images
        ├── Numeric cells → TrOCR-Base (334M params)   CER: 0.000765
        └── Text cells    → TrOCR-Large (558M params)  CER: 0.000765
```

Splitting training data by content type (numeric vs. text) narrows each model's output vocabulary, improving accuracy over a single general-purpose student.

## Results

| Model | Params | vs. Teacher (active) | CER |
|---|---|---|---|
| Qwen3-VL-235B (teacher) | 235B (22B active) | — | — |
| TrOCR-Base (numeric) | 334M | 66× smaller | **0.000765** |
| TrOCR-Large (text) | 558M | 39× smaller | **0.000765** |

## Cost Profile

| Phase | Cost |
|---|---|
| Labeling (one-time) | Fixed — Bedrock Batch at $0.53/1M input + $2.66/1M output tokens |
| Inference (recurring) | ~$0.08–$0.17/day (5–10 min/day on `ml.g5.xlarge`) |
| Annual inference | ~$30–$60/year |

Results are cached, so the model only processes new or changed images each day.

## Training

- **Infrastructure**: `ml.g5.12xlarge` (4× A10G GPUs, AWS SageMaker)
- **Framework**: PyTorch DDP (NCCL)
- **Epochs**: 10 | **Batch size**: 128 (32 per GPU) | **LR**: 5e-5 (AdamW + linear warmup)
- **Mixed precision**: AMP with GradScaler

## Distillation Type

This uses **data-augmentation-based distillation** — the teacher generates hard labels (exact transcriptions), and the student trains on them with standard cross-entropy loss. No access to teacher logits or intermediate representations is required. This pattern applies to any task where a large VLM can reliably label unlabeled data at scale.
